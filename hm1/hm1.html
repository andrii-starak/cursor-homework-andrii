• git config --global user.name blabla - підпис користувача який буде вносити зміни. Важливо в командній роботі для відстеження роботи і правок кожного члена команди. Необхідне для роботи багатьох команд.
  • git config --global user.email blabal@bla - прикріплення емейлу для підтвердження входу та звязку із Github. Global - говорить про те що це налаштування глобального рівня, і всі подальші проекти будуть підтягувати введені параметри поточного користувача.
  • git init - створення пустого репозиторію в папці кореня .git
  • git status - перевіряє стастус файлів проекту (чи знаходиться файл на проміжному етапі перед комітом, чи були внесені зміни)
  • git add - підготовка файлу до коміту, додавання його файлу в проміжний етап (індексування). Виконується після редагування файлу.
  • git commit - всі підготовленні файли (проіндексовані) зберігає у внутрішньому сховищі. (з командою -m одразу можна додати коментар, до версії яка була закомічена)
  • git log - перечислення всіх зроблених комітів в репозиторії та інформації про їх автора: імя, пошта, час коміту тощо.
  • git checkout - переключення між гілками проектів
  • git branch - менеджер гілок в Git. інструмент за допомогою якого можна перейменовувати, перечисляти, видаляти і створювати гілки.
  • git push - відправлення всіх закомічениї файлів на дистанційний репозиторій
  • git pull - обєднує зміни файли з віддаленого репозиторію і обєднує з локальними файлами. (Отримуєш останню версію яка була на репозиторії)
  • git merge - приєднання гілки до активної гілки
  • git clone - клонує репозиторій (копіює файли)
  • .gitignore файл - це тектовий файл в якому вказується, які файли або папки повинен ігнорувати Git і не передавати у віддалений репозиторій.
  • git revert- створення нового коміту з відкатом попередніх комітів (версія яка була декілька комітів назад)

  • git reset - для скидування індекса (після git add) і скидання робочого каталогу до версії останнього коміту який був зроблений. всі зміни після останнього коміту позначаються як невідслідковані
  • git rebase - перемещает историю всей ветки поверх другой, переписывая историю проекта новыми коммитами. Використовуєтсья для чистішої гілки проекту.
  • git cherry-pick - зміни внесені одним комітом, застосовуються повторно у вигляді нового коміту в поточну гілку
  • git stash - тимчасове збереження непідтвердженних змін в локальному репозиторії, для виконання іншої роботи повязаної з іншими гілками, комітами тощо. При виконанні push - інформація зі стеша не записується на віддаленому репозиторії.
